# Лабораторна робота 5: Expo Router + Auth + Каталог

Мобільний застосунок на Expo Router з авторизацією, захищеними маршрутами та екраном деталей товару.

## Що реалізовано

- Вхід та реєстрація користувача з базовою валідацією полів.
- Захищена група маршрутів через Auth Guard.
- Каталог товарів на FlatList.
- Динамічна сторінка деталей товару за id.
- Вихід із системи через глобальний AuthContext.
- Кастомний екран для неіснуючих маршрутів.

## Структура проєкту

```text
app/
    _layout.js
    +not-found.js
    (auth)/
        login.js
        register.js
    (app)/
        _layout.js
        index.js
        details/
            [id].js
context/
    AuthContext.js
data/
    products.js
```

## Запуск проєкту

1. Встановити залежності:

```bash
npm install
```

2. Запустити застосунок:

```bash
npx expo start
```

3. Відкрити в Expo Go або емуляторі.

## Контрольні запитання

1. Перенаправлення неавторизованого користувача в Expo Router реалізується через захищений layout. У файлі групи маршрутів перевіряється стан авторизації, і якщо користувач не увійшов у систему, компонент `Redirect` автоматично переводить його на екран входу.

2. Компонент `Link` використовується для декларативної навігації, коли перехід описується прямо в розмітці. Метод `router.push()` застосовується імперативно всередині функцій, наприклад після обробки події або успішної валідації.

3. Динамічні маршрути в Expo Router створюються через назву файлу в квадратних дужках, наприклад `[id].js`. Передані параметри отримуються через хук `useLocalSearchParams()`, який повертає значення з URL.

4. Стан авторизації зручно зберігати у глобальному контексті React Context, тому що він потрібен багатьом екранам одночасно. Це дозволяє уникнути prop drilling і не дублювати логіку в локальному стані окремих компонентів.

5. Групи маршрутів, наприклад `(auth)` або `(app)`, використовуються для логічного розділення екранів і різних layout-структур. Такі папки не додаються до URL-адреси, а служать лише для організації навігації та макетів.
