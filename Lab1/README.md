# FirstMobileApp 

**Застосунок складається з трьох основних екранів:**
1.  **Головна:** Список новин із заголовками, датами та коротким описом.
2.  **Фотогалерея:** Сітка (grid) з порожніх карток, що демонструє роботу зі списками.
3.  **Профіль:** Форма реєстрації користувача (Email, Пароль, Прізвище, Ім'я).

## Інструкція із запуску

Для запуску проєкту на локальній машині виконайте наступні кроки:

1.  Клонуйте репозиторій та перейдіть у папку проєкту:
    ```bash
    cd lab1
    ```
2.  Встановіть необхідні залежності:
    ```bash
    npm install
    ```
3.  Запустіть сервер розробки Expo:
    ```bash
    npx expo start
    ```
    *Або для запуску через тунель:*
    ```bash
    npx expo start --tunnel
    ```
4.  Відскануйте QR-код за допомогою додатка **Expo Go** на вашому смартфоні

## Основні способи запуску мобільного додатка

Під час розробки та тестування React Native застосунків через Expo існують різні методи запуску, кожен з яких має свої особливості:

### 1. Реальний фізичний пристрій (через Expo Go)
* **Призначення:** Тестування продуктивності та взаємодії з реальним обладнанням.
* **Особливості:** Дозволяє перевірити жести, сенсорний ввід, роботу камери та інших датчиків у реальних умовах.
* **Відмінності:** Найточніший спосіб зрозуміти, як користувач буде відчувати додаток. Потребує однакової Wi-Fi мережі або використання тунелю.

### 2. Емулятор Android / Симулятор iOS
* **Призначення:** Швидке тестування інтерфейсу під час написання коду без потреби в руках тримати телефон.
* **Особливості:** Запускається безпосередньо на комп'ютері розробника.
* **Відмінності:** Емулятори споживають багато ресурсів ПК. Не завжди ідеально точно відтворюють роботу сенсорів фізичного пристрою.

### 3. Запуск у Web-браузері
* **Призначення:** Швидка перевірка логіки та верстки.
* **Особливості:** Не потребує мобільних пристроїв чи важких емуляторів.
* **Відмінності:** Багато "нативних" функцій (наприклад, деякі датчики або специфічні анімації) можуть не працювати або працювати інакше, ніж на мобільних ОС.

### 4. Режим тунелю (`--tunnel`)
* **Призначення:** Запуск, коли комп'ютер та телефон знаходяться у різних підмережах (наприклад, мобільний інтернет на телефоні та офісний Wi-Fi на ПК).
* **Особливості:** Трафік проходить через сервери Expo.
* **Відмінності:** Працює повільніше за пряме з'єднання, але є незамінним у складних мережевих умовах
